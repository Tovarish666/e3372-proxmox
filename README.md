# e3372-proxmox

Драйвер-инсталлер для модемов **Huawei E3372 (HiLink)** на хосте **Proxmox VE / Debian**.
Одна команда — и пачка модемов поднимается, маршрутизируется и готова к работе.

Проверено на E3372**h-153**; те же шаги подходят для **h-607, s-153, K5160** (одно семейство Balong,
композиция HiLink). Новый Qualcomm-ревиз **h-320/-325** с завода уже HiLink — переключать не нужно,
достаточно сетевого слоя.

## Установка (одной командой)

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Tovarish666/e3372-proxmox/main/install.sh)
```

~3 минуты, идемпотентно (можно повторять). Требует root.

## Что делает

| Шаг | Действие |
|-----|----------|
| 1 | Ставит `usb-modeswitch (+data)`, `networkd-dispatcher` |
| 2 | Композиция HiLink: модем из CD-ROM (`12d1:1f01`) → сетевая карта (`12d1:14dc`) |
| 3 | Поднимает интерфейсы модемов по DHCP через **systemd-networkd** (матч по драйверу, не по имени/MAC — у всех E3372 одинаковый MAC) |
| 4 | Вешает **per-modem policy routing**: своя таблица на модем, `ip rule from 192.168.N.100 → default via 192.168.N.1` — авто на boot и hotplug |
| 5 | Loose `rp_filter` (иначе ответы на 20 интерфейсах режутся) |
| 6 | Ставит диагностику `e3372-check` |

## Модель адресации

Каждый модем настраивается (на ноуте, заранее) на **свою подсеть**:

```
модем N:  192.168.N.0/24    шлюз/web-API 192.168.N.1    хост получает 192.168.N.100
```

Хост биндит исходящий трафик на `192.168.N.100`, policy routing гонит его через `192.168.N.1`.
Это ровно то, что ждёт [modlink](https://github.com/Tovarish666/modlink) (`inet4_bind_address: 192.168.N.100`).

> ⚠️ **Коллизия подсетей.** Если подсеть модема совпадает с сетью самого хоста
> (напр. модем в `192.168.88.x`, а `vmbr0` тоже `192.168.88.0/24`), маршруты **не создаются**
> для этого модема — иначе шлюзы и ARP пересекутся. Такой модем инсталлер **авто-пропускает**
> и пишет в лог. Перенастрой его на свободную подсеть (напр. `192.168.101.x`).

## Проверка

```bash
e3372-check
```
```
N    IFACE            HOST-IP          WEBUI  CONN      NET   EXIT-IP
81   eth17            192.168.81.100   ok     онлайн    19    100.x.x.x   <- сотовый IP = работает
82   eth14            192.168.82.100   ok     онлайн    19    100.x.x.x
...
```

- `WEBUI=ok` — модем жив и отвечает по API (маршрутизация для этого не нужна, адрес on-link).
- `EXIT-IP` = сотовый адрес — трафик реально выходит через модем end-to-end.
- `EXIT-IP=—` при `CONN=онлайн` — проблема с policy routing; `EXIT-IP=—` при `CONN=offline/нет-сети` — APN/сигнал/SIM.

## Что ставится в систему

```
/etc/systemd/network/25-hilink.network          # DHCP на модемных интерфейсах
/etc/sysctl.d/99-e3372.conf                      # rp_filter=2
/etc/networkd-dispatcher/routable.d/50-e3372     # policy routing hook
/usr/local/bin/e3372-check                       # диагностика
```

## Удаление

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Tovarish666/e3372-proxmox/main/uninstall.sh)
```

## Заметки

- systemd-networkd работает **параллельно** с ifupdown2 Proxmox: `.network` матчит только модемы
  по драйверу, `enp*`/`vmbr*` не трогает.
- Переключение из CD-ROM в сеть делает прошивка модема или `usb-modeswitch`. Перепрошивку
  stick→HiLink (если у модема stick-firmware) делай на ноуте отдельно — это вне scope хоста.
- Смена IP модема (реконнект) — задача HiLink API самого модема / панели modlink, не этого слоя.
